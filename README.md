# Реализация абстрактных структур данных

```markdown
Этот проект реализует различные абстрактные структуры данных с базовым набором операций и обеспечивает удобный интерфейс для работы с ними.

## Структуры данных

- **Массив**: реализация массива с операциями добавления, получения, удаления, замены элементов и определения длины.
- **Список**: односвязный и двусвязный списки с операциями добавления, удаления, поиска и чтения элементов.
- **Очередь**: реализация очереди с операциями добавления, удаления и чтения элементов.
- **Стек**: реализация стека с операциями добавления, удаления и чтения элементов.
- **Хеш-таблица**: хеш-таблица с обработкой коллизий, операциями добавления, получения и удаления элементов.
- **АВЛ-дерево**: сбалансированное двоичное дерево с операциями добавления, поиска, удаления и чтения.

## Интерфейс и работа с файлами

Для взаимодействия со структурами данных используется интерфейс командной строки. Запуск программы осуществляется следующей командой:

\`\`\`bash
./dbms --file <имя_файла> --query <запрос>
\`\`\`

Где:
- \`--file <имя_файла>\` - путь к файлу с данными.
- \`--query <запрос>\` - запрос к базе данных, включающий операцию и параметры.

Примеры запросов:

\`\`\`bash
./dbms --file data.txt --query 'SPUSH mystack item'
./dbms --file data.txt --query 'QPOP myqueue'
./dbms --file data.txt --query 'HSET myhash key value'
\`\`\`

## Сложность операций (Big O notation)

| Структура данных | Добавление | Удаление | Поиск | Чтение |
|------------------|------------|----------|-------|--------|
| Массив           | O(1)       | O(n)     | O(1)  | O(n)   |
| Список           | O(1)       | O(n)     | O(n)  | O(n)   |
| Очередь          | O(1)       | O(1)     | O(n)  | O(n)   |
| Стек             | O(1)       | O(1)     | O(n)  | O(n)   |
| Хеш-таблица     | O(1)       | O(1)     | O(1)  | O(n)   |
| АВЛ-дерево       | O(log n)   | O(log n) | O(log n) | O(n) |

## Заключение

Данный проект является полезным инструментом для изучения и сравнения различных структур данных, а также может быть использован в качестве основы для более сложных систем управления данными.
```
